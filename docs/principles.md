# 開発原則と未解決課題

## 開発上の決めごと

- **Degrade 最優先**: 拡張に依存しない最小構成を常に維持し、追加機能は段階的に載せる。
- **メタデータ先行**: 文字列・依存・近接の 3 軸で 80% のニーズを満たし、Embedding は補完に留める。
- **再現性重視**: 同一入力に対してランキング結果が決定論的になるよう乱数シードを固定する。

## オープンな論点 / リスク

- **TypeScript モジュール境界**: モノレポでの `common/` 誤命中を防ぐ規約設計が必要。
- **埋め込み更新コスト**: 差分のみ再計算するか、低頻度のバッチ更新で妥協するか判断が必要。
- **巨大バイナリ流入**: 自動除外・検知の閾値やワークフローを継続的に改善する必要がある。
